My UX and UI Projects

I have taken multiple courses on website development (backend and frontend). Here are some of the projects that I have contributed to the UX and UI design of, as well as the development and deployment of them.

Travelz:

Travelz allows a group of people traveling together to collaboratively build their shared travel journal/story. Others can search for and view the story as it unfolds. It allows all co-travelers to add new entries (containing text and images) to a shared journal. Entries are marked with the login name of the creator and timestamp of creation. As more and more entries are created, the group sees their shared memories unfold, and preserved for years to come. Complete UX and UI design of the website was created by the project team I was in. Development and publishing was also done by the same team.

Languages and Tools Implemented:
  1. HTML
  2. JavaScript
  3. CSS
  4. Python (Using the following packages):
          - Flask
          - Gunicorn
          - Authlib
  5. PostgreSQL
  6. QuillJS as markdown editor

Key Features:

1. Dynamic scrolling, such that initially only six journals load, with more following as the user scrolls to the bottom of the page.
2. Dynamic editing of created entries, where the toolbar displays only when clicking inside of the entry box and it updates automatically when the user clicks outside of it.
3. Formatting the “Add Contributors” button such that a dropdown is displayed with the users' nicknames that correspond to what the user has typed in–as well as the tooltip displayed above the input.
4. All styling in general; of particular note is the home page’s grid-like setup with the text fading to signify that the title and/or the description is too long to fit in the journal overview display.

Recipes:

A recipes page that will allow for the creation of a recipe list: You can enter recipes that you have made in the past or want to make in the future, and they will all be displayed in one convenient place. Each time you cook a recipe, you can log that in as an “attempt”, with its own rating on how it turned out, and any notes of things you could change in the future. You can filter your recipes by “wishlist” (to view untested recipes), number of times made, rating, etc. Complete UX and UI design of the website was created by the project team I was in. Development and publishing was also done by the same team.

Languages and Tools Implemented:
  1. HTML
  2. JavaScript
  3. CSS
  4. Vue (Using the following dependencies):
          - Material
          - Quill Editor
          - Star Rating
          - Autosuggest
  5. Firebase

Key Features:

1. Public and private recipes and allowing any user to add/copy a public recipe to their own personal list.
2. Each recipe has a list of attempts, where it’s total rating is calculated as the average of the attempts' ratings. The uploaded image of the highest rated attempt is featured on that recipe’s page.
3. Filtering and sorting are placed into a component which was used separately in both public and personal recipe lists. The search filter will autocomplete searches with keywords from both titles and ingredient lists, and include multiple words for turning “brow” into both “brown sugar” and “brown rice”, etc.
4. The mobile camera device integration is supported to allow users to take pictures of their food using their phone.
5. As a PWA, this WebApp can be saved on the home screen of a smart phone.

Tavolo:

A mobile app for dine in restaurants. Allows you to reserve your table at a restaurant and pre-order your food directly from the app. Allows groups to create orders separately as well as payment through the app. Complete UX and UI design of the app was created by a design team under my leadership.

Key Features:

1. Making a reservation as well as ordering your food, with optional payment in app.
2. Adding friends to reservation to allow them to place and pay for their own order on their phone.
3. Map search and filter function.
4. User rating for each restaurant.

Connect2Play Sports:

A platform similar to a social network and a sport’s group locating tool at the same time. Once you sign up, you can find a sporting group, a team that needs players, or create your own pickup game within your area with a few clicks. Just select the sports and fitness activities you are interested in, the location of the event you want to play, and any other parameters (such as level of play and time of event). Connect2Play Sports covers a diverse list of sports. Everything from archery, basketball, football, broomball, kayaking, canoeing, darts, canyoneering, cycling, curling, dodgeball, paintball, pickleball, bowling, lawn game, weight lifting, hockey, tennis, yahtzee, yoga, to even casual walking. Complete UX and UI design of the website was created by a design team I was in. Development and publishing was done by myself and a senior developer.

Languages and Tools Implemented:
  1. HTML
  2. JavaScript
  3. CSS
  4. PHP
  5. MySQL

Key Features:

1. Account creating and authentication
2. Map integration to allow users to place and locate events on a map.
3. Two different grouping types: organizations and teams. Organizations (eg. 3M) can have multiple teams in them and create leagues for teams to sign up for.
4. User statistic tracking to allow users to see how well they perform in matches as well as check the integrity and consistency of other users showing up to events.
