Recipes

Concept:

A recipes page that will allow for the creation of a recipe list: You can enter recipes that you have made in the past or want to make in the future, and they will all be displayed in one convenient place. Each time you cook a recipe, you can log that in as an “attempt”, with its own rating on how it turned out, and any notes of things you could change in the future. You can filter your recipes by “wishlist” (to view untested recipes), number of times made, rating, etc. Complete UX and UI design of the website was created by the project team I was in. Development and publishing was also done by the same team.

Languages and Tools Implemented:
  1. HTML
  2. JavaScript
  3. CSS
  4. Vue (Using the following dependencies):
          - Material
          - Quill Editor
          - Star Rating
          - Autosuggest
  5. Firebase

Key Features:

1. Public and private recipes and allowing any user to add/copy a public recipe to their own personal list.
2. Each recipe has a list of attempts, where it’s total rating is calculated as the average of the attempts' ratings. The uploaded image of the highest rated attempt is featured on that recipe’s page.
3. Filtering and sorting are placed into a component which was used separately in both public and personal recipe lists. The search filter will autocomplete searches with keywords from both titles and ingredient lists, and include multiple words for turning “brow” into both “brown sugar” and “brown rice”, etc.
4. The mobile camera device integration is supported to allow users to take pictures of their food using their phone.
5. As a PWA, this WebApp can be saved on the home screen of a smart phone.
